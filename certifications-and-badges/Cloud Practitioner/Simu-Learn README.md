# AWS SimuLearn: Cloud Practitioner

## Overview

I completed the comprehensive AWS Cloud Practitioner SimuLearn program, a hands-on series of labs designed to build a solid foundation in AWS cloud concepts and services. Through twelve focused SimuLearns, I gained practical experience architecting scalable solutions, implementing networking best practices, optimizing costs, and leveraging core AWS services to solve real-world business problems.

## Services I Worked With

* **Amazon Elastic Compute Cloud (EC2)** — Deployed and managed scalable compute instances across multiple Availability Zones, configured instance types for different workload requirements, and implemented connection methods for secure access
* **Amazon EC2 Auto Scaling** — Implemented dynamic scaling policies, configured health checks and auto-healing capabilities, and designed scaling strategies responding to both metric-based and time-based demand patterns
* **Elastic Load Balancing (ELB)** — Analyzed Application Load Balancer capabilities, configured load distribution across instances, and implemented health monitoring to ensure traffic routing to healthy targets
* **Amazon Simple Storage Service (S3)** — Implemented static website hosting, stored application assets, and evaluated storage options for different use cases
* **Amazon VPC (Virtual Private Cloud)** — Designed isolated network environments with custom subnets, routing, and peering connections; implemented security controls through NACLs and security groups; and configured traffic routing with internet gateways and NAT gateways
* **Amazon Relational Database Service (RDS)** — Deployed and managed relational databases with Multi-AZ high availability, configured read replicas for performance optimization, and evaluated database migration strategies
* **Amazon DynamoDB** — Designed NoSQL table structures, optimized for performance and cost, and implemented flexible schema designs for application requirements
* **Amazon Elastic File System (EFS)** — Configured centralized file system access across multiple EC2 instances with high availability and shared storage capabilities
* **AWS CloudWatch** — Monitored application metrics, set up alarms, and triggered scaling actions based on performance indicators
* **AWS Identity and Access Management (IAM)** — Managed users, roles, and policies to control secure access to AWS resources
* **AWS Database Migration Service (DMS)** — Assessed and executed database migrations from on-premises infrastructure to AWS
* **AWS Pricing Calculator** — Built cost models for multi-service architectures and optimized cloud spending

## What I Demonstrated

* **Cloud Fundamentals** — Articulated core characteristics of AWS cloud computing and analyzed key benefits including scalability, flexibility, and cost efficiency compared to on-premises infrastructure
* **Global Infrastructure & High Availability** — Leveraged AWS's distributed global architecture, Regions, and Availability Zones to design resilient solutions with fault tolerance across multiple geographic locations
* **Compute Solutions** — Compared EC2 instance families and types, evaluated vertical and horizontal scaling strategies, and matched instance selection to specific workload requirements
* **Auto Scaling & Resilience** — Implemented auto-healing capabilities, configured metric-based and time-based scaling policies, and designed Auto Scaling groups with resource boundaries for cost control
* **Load Balancing & Distribution** — Analyzed Application Load Balancer functionality, implemented health checks, and configured traffic distribution across multiple instances for high availability
* **Networking & VPC Architecture** — Designed VPC architectures with subnets, gateways, and routing controls; applied CIDR notation for IP addressing; implemented peering connections between VPCs; and configured security controls through NACLs and security groups
* **Database Solutions** — Compared relational and NoSQL database types, implemented Multi-AZ deployments for high availability, configured read replicas for scaling, and evaluated database migration strategies
* **Storage Options** — Evaluated and applied different AWS storage solutions including block storage (EBS), object storage (S3), and file systems (EFS) for appropriate use cases
* **Security & Access Control** — Implemented IAM policies and roles following principle of least privilege, understood the AWS Shared Responsibility Model, and configured secure access to resources
* **Cost Optimization** — Used AWS Pricing Calculator to estimate costs, analyzed cost optimization strategies including instance selection and resource right-sizing, and evaluated trade-offs between performance, availability, and expense
* **Infrastructure as Code & Configuration** — Configured launch templates, route tables, security group rules, and automated deployments across distributed infrastructure
