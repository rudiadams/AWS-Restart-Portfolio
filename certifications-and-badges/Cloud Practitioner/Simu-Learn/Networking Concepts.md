# Networking Concepts

## Overview
My exploration of AWS networking fundamentals — analyzing the core components of Virtual Private Clouds, designing network architectures with subnets and gateways, applying IP addressing and CIDR notation, and configuring traffic routing through security controls and network appliances.

---

## Services I Worked With

- **Amazon VPC** — virtual networking environment that provides isolated, configurable network infrastructure with subnets, gateways, and routing controls
- **Amazon Elastic Compute Cloud (EC2)** — deployed within VPC architecture to validate network connectivity and traffic routing configurations

---

## What I Demonstrated

Through this SimuLearn, I demonstrated knowledge of:

- VPC architecture and core networking components including subnets, internet gateways, and route tables
- The advantages and design benefits of VPC implementations for network isolation and security
- CIDR block notation and IP addressing concepts for network planning and allocation
- VPC traffic routing configuration using internet gateways, NAT gateways, and route table rules
- Network Access Control Lists (NACLs) for stateless traffic filtering
- Security groups for stateful firewall control at the instance level
- How networking layers work together to control inbound and outbound traffic flows
