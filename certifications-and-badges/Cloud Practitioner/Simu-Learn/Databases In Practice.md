# Databases in Practice

## Overview
My deep dive into AWS database solutions — comparing database types and their use cases, implementing Multi-AZ deployments for high availability, configuring read replicas for performance optimization, and evaluating database migration strategies using AWS services.

---

## Services I Worked With

- **Amazon Relational Database Service (RDS)** — fully managed relational database service supporting multiple engines with high availability, read scaling, and automated backup capabilities
- **AWS Database Migration Service (DMS)** — service for assessing and migrating databases from on-premises or other cloud platforms to AWS

---

## What I Demonstrated

Through this SimuLearn, I demonstrated knowledge of:

- Database types available on AWS and their appropriate use cases
- Relational database architecture and design principles
- Multi-AZ deployment implementation for database high availability and fault tolerance
- Read replica configuration and deployment for scaling read-heavy workloads
- Performance optimization strategies through replica placement and replication lag management
- Managed versus hosted database solution trade-offs and architecture decisions
- Database migration assessment, planning, and execution using AWS DMS
- Migration strategies including full load, ongoing replication, and cutover approaches
- Data consistency and downtime considerations during database migrations
