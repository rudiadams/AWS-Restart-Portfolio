# Core Security Concepts

## Overview
My introduction to AWS security fundamentals — comparing IAM identity models, analyzing policy structures, understanding the Shared Responsibility Model, and implementing security best practices for managing secure access to AWS resources.

---

## Services I Worked With

- **AWS Identity and Access Management (IAM)** — service for managing users, roles, groups, and policies to control access to AWS resources with fine-grained permissions
- **Amazon Elastic Compute Cloud (EC2)** — compute service configured with IAM roles for secure resource access
- **Amazon Relational Database Service (RDS)** — database service secured through IAM authentication and access policies

---

## What I Demonstrated

Through this SimuLearn, I demonstrated knowledge of:

- IAM identity types including users, roles, and groups, and their creation and management
- Use cases and differences between users, roles, and groups for access control
- IAM policy structure including principals, actions, resources, and conditions
- Policy evaluation logic and permission boundaries
- The AWS Shared Responsibility Model and division of security responsibilities between AWS and customers
- AWS compliance programs and their relevance to organizational security requirements
- IAM best practices for secure access management and principle of least privilege
- Role-based access control for applications and cross-service resource access
- Security implications of identity and access management decisions
